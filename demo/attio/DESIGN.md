# Overview

This homepage is a long-form SaaS marketing page built around demonstrating a product interface rather than telling a brand story through imagery.

The dominant visual rule is:

**Large whitespace + thin structural grids + product UI demonstrations + restrained typography.**

The page feels engineered rather than designed for emotional impact. Nearly every section exists to explain product capabilities, workflows, infrastructure, integrations, or outcomes.

Visual hierarchy is created through:

- Large bold headlines
- Extremely restrained color usage
- Thin borders
- Grid systems
- Product screenshots
- Diagrammatic illustrations
- White space separation

Photography appears only later in the page and is used sparingly.

The homepage alternates between:

1. Narrative sections
2. Product demonstrations
3. Workflow explanations
4. Customer proof
5. Infrastructure proof
6. Closing conversion sections

The overall atmosphere is:

- Technical
- Structured
- Confident
- Product-led
- Modern
- Minimal
- Systematic

Not playful.
Not heavily branded.
Not illustration-driven.
Not visually loud.

# Key Characteristics

- Predominantly white background.
- Very limited accent color usage.
- Heavy reliance on black typography.
- Large centered hero.
- Product UI is the primary visual asset.
- Repeated use of faint grid overlays.
- Hairline dividers throughout the page.

- Screenshot-first storytelling.
- Multiple workflow diagrams.
- Dense CRM-style interface examples.
- Minimal shadows.
- Rounded rectangles used consistently.

# Do / Don't

## Do

- Preserve whitespace.
- Preserve grid structure.
- Preserve product-first storytelling.
- Use restrained color.
- Use hairline borders.
- Keep typography dominant.
- Use interface screenshots as hero assets.

## Don't

- Add heavy gradients.
- Add oversized shadows.
- Add decorative illustrations everywhere.
- Use bright marketing colors as page backgrounds.
- Use excessive animations.
- Break grid alignment.
- Introduce skeuomorphic UI.
